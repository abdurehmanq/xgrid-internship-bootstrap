## ðŸš€ Git Workflow Followed

This repository follows a professional Git workflow:

1. Work is performed in feature branches.
2. Changes are committed using structured commit messages.
3. A Pull Request (PR) is created.
4. Changes are reviewed.
5. PR is merged into the `main` branch.

### Example Branch Naming

- `feature-update-readme`
- `feature-add-health-report-script`

### Example Commit Format
feat(script): add health report script
docs(readme): update internship workflow section

## Contributing
